# 05. 스터디 가이드

## 1) 학습 목표

- 0~4단계 로드맵을 기능/리스크/완료 조건 관점으로 설명한다.
- API/아키텍처/테스트 문서를 연결해서 이해한다.
- 4인 협업 역할에 맞춰 발표/리뷰/회고를 수행한다.

## 2) 권장 학습 운영(팀 4인)

- 단계 시작 전 30분: 목표/리스크/테스트 포인트 정렬
- 단계 종료 후 20분: 결과 회고(성공 기준 충족 여부 확인)
- 산출물: 단계 요약 1페이지 + 다음 단계 질문 3개

## 3) 0~4단계 학습 로드맵

| 단계 | 핵심 학습 주제 | 성공 기준 | 리스크 | 완료 조건 |
| --- | --- | --- | --- | --- |
| 0 | 공통 용어, 문서 구조, 규칙 | 팀 공통 정의 사용 | 용어 혼선 | 용어/규칙 합의 |
| 1 | KV 명령과 오류 처리 | 입력/출력 계약 설명 가능 | 정상 흐름만 이해 | 명령별 예시 3개 정리 |
| 2 | TTL/만료 경계 | `-2/-1/>=0` 규칙 설명 가능 | 시간 로직 오해 | 경계 시나리오 노트 작성 |
| 3 | prefix 무효화/운영 지표 | 무효화 위험과 보호 규칙 설명 | 과삭제 인식 부족 | 운영 체크포인트 합의 |
| 4 | 부하 결과 해석/릴리스 판단 | 성능 지표로 의사결정 가능 | 수치 해석 부족 | 최종 리포트 리뷰 완료 |

## 4) 4인 역할 분담(학습 관점)

| 역할 | 책임 |
| --- | --- |
| PM | 세션 아젠다, 일정, 회고 기록 |
| TL | 기술 주제 정리, 설계 질의 대응 |
| DEV | 구현 관점 사례 공유, 트러블슈팅 |
| QA | 테스트 관점 설명, 게이트 해석 |

## 5) 테스트 분류 학습 포인트

- Unit: 함수/모듈 단위 규칙 검증
- Integration: API부터 저장소까지 흐름 검증
- Failure: 잘못된 입력/경계/오류 처리 검증
- Load: 처리량/지연/오류율 기반 성능 검증

## 6) 세션 체크리스트

- 오늘 단계 성공 기준을 먼저 읽었는가?
- 리스크 대응안을 최소 1개 도출했는가?
- Unit/Integration/Failure/Load 관점으로 설명했는가?
- 문서 변경 필요 항목을 명시했는가?
